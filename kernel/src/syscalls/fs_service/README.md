# Filesystem System Calls Module

## 概述

此模块包含所有与文件系统相关的系统调用实现，包括文件操作、目录管理、文件权限控制等功能。

## 系统调用范围

### 文件操作 (0x2000-0x2FFF)
- `open()` - 打开或创建文件
- `read()` - 从文件读取数据
- `write()` - 向文件写入数据
- `close()` - 关闭文件描述符
- `ioctl()` - I/O控制操作
- `fstat()` - 获取文件状态信息
- `lseek()` - 移动文件读写位置

### 目录操作
- `mkdir()` - 创建目录
- `rmdir()` - 删除目录
- `chdir()` - 更改当前工作目录
- `getcwd()` - 获取当前工作目录
- `readdir()` - 读取目录内容

### 文件系统管理
- `mount()` - 挂载文件系统
- `umount()` - 卸载文件系统
- `stat()` - 获取文件系统统计信息
- `chmod()` - 更改文件权限
- `chown()` - 更改文件所有者

### 高级I/O操作
- `sendfile()` - 零拷贝文件传输
- `splice()` - 数据管道操作
- `tee()` - 数据复制操作

## 相关文件

- `fs.rs` - 基础文件系统系统调用
- `file_io.rs` - 文件I/O操作系统调用
- `zero_copy.rs` - 零拷贝优化功能
- `aio.rs` - 异步I/O系统调用
- `mod.rs` - 模块声明和导出

## 架构依赖

依赖以下内核子系统：
- `crate::vfs` - 虚拟文件系统抽象层
- `crate::fs` - 具体文件系统驱动
- `crate::mm::vm` - 虚拟内存管理（零拷贝）
- `crate::process::manager` - 进程权限上下文

## 重构目标

根据分析报告，此模块应该独立成为一个服务，提供文件系统的统一接口，减少与其他模块的耦合，提高性能并增强可维护性。计划支持多重文件系统类型，通过抽象接口实现灵活扩展。
